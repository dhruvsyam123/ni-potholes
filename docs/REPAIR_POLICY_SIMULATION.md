# Repair Policy Simulation

This is the clearest answer to the operational question:

> If we forecast recurrence risk and repair high-risk defects differently, what
> percentage cost saving could we plausibly get?

The model is trained on observed official pothole-defect recurrence. The repair
effect is not directly trained because the open data does not contain repair
date, repair type, unit cost, repair quality, or randomized intervention
outcomes.

So this is a counterfactual simulation, not causal proof.

## Simulation Logic

Baseline:

```text
Every observed holdout defect occurs and costs event_cost.
```

Policy:

```text
For each defect in chronological order:
  if the trained model score >= threshold:
    pay an intervention premium
    suppress future defects within 200 m for N days with probability effectiveness
```

This creates a dynamic effect: repairing one high-risk defect can prevent later
nearby defects, which then also means fewer later repair decisions.

The simulation runs on the true future holdout:

```text
2017-10-01 to 2018-03-17
49,360 official pothole defects
```

## Recommended Threshold

For a practical pilot, use:

```text
risk >= 0.35 to 0.40
```

This is lower than the earlier static cost-policy threshold because the dynamic
simulation accounts for downstream defects being prevented. A high threshold
like `0.70` has better precision but misses many early intervention
opportunities.

Suggested deployment policy:

```text
Default fast/high-quality repair: risk >= 0.40
If crews have spare capacity:     risk >= 0.35
If intervention is very cheap:    risk >= 0.30
If intervention is very expensive: risk >= 0.70+
```

## What Percentage Saving Should We Quote?

Use ranges, not a single number.

Defensible short-term authority-cost range:

```text
~14% to 15% saving
```

This comes from repair-only or authority/admin scenarios with 14-day suppression.

Plausible asset-management range if better repairs suppress recurrence for
30-90 days:

```text
~18% to 23% saving
```

Stress-test lower bound for expensive patching:

```text
~2% saving
```

That scenario treats a medium patch premium as very large. It shows that major
patching should be targeted narrowly or justified using broader evidence than
14-day recurrence alone.

## Important Caveat

This analysis answers:

```text
What would have happened if high-risk observed defects had been given a repair
that probabilistically suppresses nearby future defects?
```

It does not prove:

```text
This exact repair method causally reduces future defects by X%.
```

To prove that, we would need actual repair logs or a pilot with treatment/control
comparison:

- defect recorded
- inspection time
- repair time
- repair type
- repair cost
- repair crew/material
- whether the same location failed again

The current model is still useful because it identifies where such interventions
should be tested first.
